# Suggested Reading

## Purpose

These recommendations target the gap between a strong Linux host-operations repository and a
world-class DevOps/SRE-grade operations engine.

## Books

## 1. *Site Reliability Engineering*

### Why it matters

This is the canonical source for:

- reliability thinking
- operational toil reduction
- alerting and monitoring philosophy
- incident response discipline

### How it would improve `linux-skills`

It would help the repo evolve from strong host administration into stronger:

- observability design
- alerting philosophy
- operational prioritization
- reliability-oriented workflows

## 2. *The Site Reliability Workbook*

### Why it matters

It turns SRE ideas into practical operating patterns.

### How it would improve `linux-skills`

It would strengthen:

- runbooks
- alert tuning
- post-incident practices
- rollout and rollback thinking

## 3. *UNIX and Linux System Administration Handbook*

### Why it matters

This remains one of the best broad references for real-world Linux operations.

### How it would improve `linux-skills`

It would deepen:

- service operations
- network and mail administration
- production conventions that go beyond one host

## 4. *The Practice of System and Network Administration*

### Why it matters

This is excellent for systems operations as a discipline, not just a technical toolbox.

### How it would improve `linux-skills`

It would improve:

- policy consistency
- environment standardization
- operational process design
- team-oriented administration

## 5. *Accelerate*

### Why it matters

It ties operational maturity to measurable outcomes.

### How it would improve `linux-skills`

It would support future growth into:

- release operations
- change safety
- recovery performance
- automation value measurement

## 6. *Terraform: Up & Running*

### Why it matters

If the repository expands from host operations into environment provisioning, IaC becomes critical.

### How it would improve `linux-skills`

It would help future work on:

- repeatable infrastructure creation
- environment parity
- safer scale

## Frameworks And Standards

## 1. SRE practices

Useful for:

- defining service reliability goals
- reducing toil
- building stronger incident workflows

## 2. GitOps / declarative operations

Useful for:

- stronger desired-state models
- environment promotion
- clearer auditability

## 3. CIS Benchmarks for Ubuntu/Debian

Useful for:

- formalizing hardening baselines
- improving security reporting
- aligning recommendations with recognized standards

## 4. NIST incident-response and recovery guidance

Useful for:

- evidence-driven incident handling
- stronger recovery discipline
- better separation of detection, containment, recovery, and lessons learned

## 5. Prometheus and observability best practices

Useful for:

- cardinality discipline
- exporter design
- better metrics and alerting hygiene

## Suggested Reading Order

1. *Site Reliability Engineering*
2. *The Site Reliability Workbook*
3. *The Practice of System and Network Administration*
4. *UNIX and Linux System Administration Handbook*
5. CIS Ubuntu/Debian Benchmarks
6. *Accelerate*
7. Terraform or GitOps material if the repo expands into broader platform operations
